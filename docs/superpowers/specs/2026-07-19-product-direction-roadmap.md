# 제품 방향 로드맵 (2026-07-19)

## 배경

지금까지 1단계(Baseline)는 완료된 상태 — 6개 도메인 API, 에이전트 포트폴리오, 배팅 정산, 라운드 생명주기, 에이전트 투자성향(AgentStyle) 분기까지 더미 데이터(코인 시드 + 합성 뉴스)로 동작 중이다. `backend/CLAUDE.md` / `frontend/CLAUDE.md`의 로드맵은 2~4단계로 "부하테스트 → 동시성 해결 → Kafka/SSE + RAG"라는 **인프라 축**을 이미 그려두고 있었다.

이 문서는 그와 별개로 **제품 축** — "최종적으로 이 서비스가 무엇을 하는 서비스인가"에 대한 방향을 정리한다. 인프라 축(부하테스트/동시성/DB튜닝)은 이 제품 축을 먼저 진행한 뒤 착수하기로 결정했다 (아래 순서 참고).

## 확정된 방향

- **투자 성격**: 실제 화폐가 오가는 실거래가 아니라, **실제 시세를 참조가로 삼는 가상 자본** 투자를 유지한다. 금융 규제 이슈를 우회한다는 기존 설계 원칙([backend/CLAUDE.md](../../../backend/CLAUDE.md))을 그대로 따른다.
- **자산군**: 코인에서 다시 주식으로 피벗한다. 실제 뉴스에 따른 가격 반응이 코인보다 뚜렷하기 때문. **한국(KRX) + 미국 주식을 동시에** 다룬다.
  - 트레이드오프: 두 시장을 합쳐도 겹치는 휴장 구간(한국 15:30~22:30경, 새벽 05:00~09:00경)과 주말은 완전히 정지한다. "쉬지 않는 시장"이 목적이 아니라 실험 가능한 자산 다양성 확보가 목적이라는 점에 합의했다.
- **뉴스**: 합성 시드 데이터 대신 실제 금융 뉴스로 교체한다.
- **인증**: `localStorage` 기반 가상 ID를 실제 로그인/회원가입으로 교체한다.
- **에이전트 개인화**: 현재는 프롬프트를 키워드 매칭해 `AgentStyle` 5종 중 하나로 1회 분류하는 수준(장식에 가까움). 이를 실제 LLM 기반 의사결정으로 고도화한다.
- **RAG 메모리**: 기존 로드맵 4단계에 있던 "과거 매매 기억을 벡터로 저장하고 유사 뉴스 발생 시 검색해 프롬프트에 주입"하는 메타인지 투자 엔진을 그대로 목표로 삼는다.

## 서브프로젝트 분해와 순서

제품 축은 서로 독립적인 5개 서브프로젝트로 쪼갠다. 각 서브프로젝트는 자체 스펙 → 계획 → 구현 사이클을 거친다.

1. **실시세 연동 (한국 + 미국 주식)** — 나머지 전부의 기반. "가격 소스"와 "에이전트 매매 시뮬레이션"을 분리하는 구조로 설계한다 (에이전트는 실제 거래소에 주문을 넣지 않고, 실시세를 참조가로 삼아 가상으로 체결).
2. **실뉴스 연동 (종목 매핑 + 호재/악재 판정)** — 실시세가 있어야 "뉴스 이후 가격이 실제로 이렇게 움직였다"가 의미를 가진다.
3. **실제 로그인/인증** — 위 둘과 독립적이고 상대적으로 가벼운 작업. 뒤의 두 항목(에이전트 지능, RAG) 전에 끼워 넣는다.
4. **에이전트 지능 고도화** — 키워드 규칙 → LLM 기반 실제 의사결정. 실뉴스가 있어야 LLM이 판단할 거리가 생긴다.
5. **RAG 메모리 엔진** — 4번의 LLM 의사결정 루프가 있어야 기억을 주입할 대상이 존재하고, 회상할 매매 기록도 어느 정도 쌓여 있어야 의미가 있다.

인프라 축(부하테스트/DB튜닝/동시성/Redis/Kafka/SSE, `backend/CLAUDE.md` 2~4단계)은 위 5개 다음으로 미룬다. 다만 각 서브프로젝트를 구현하는 과정에서 부하/동시성/인덱싱 이슈가 자연히 드러나는 지점이 생기면, 그 시점에 사용자를 불러 직접 테스트하게 한다 — 이 프로젝트의 진짜 목적이 그 병목을 스스로 겪고 해결하는 것이기 때문이다.

## 알려진 외부 의존성 (사용자 확보 필요)

각 서브프로젝트 진행 시 아래 항목은 이 세션이 대신 생성할 수 없는 외부 계정/키다 — 해당 서브프로젝트 스펙 문서에서 구체적으로 다시 짚는다.

- 1번(실시세): 한국 실시간 시세 API(예: 한국투자증권 KIS Developers), 미국 실시간 시세 API(예: Alpaca/Polygon) 계정 및 API 키.
- 2번(실뉴스): 금융 뉴스 API(예: 네이버 뉴스 API, NewsData.io 등) 계정 및 API 키.
- 4번(에이전트 지능): LLM API 키(Anthropic API 등).
